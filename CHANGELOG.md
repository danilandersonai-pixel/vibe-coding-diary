# Changelog

Формат близок к Keep a Changelog. Версии пока проектные, без Git-тегов.

## 0.2.0 - 2026-05-17

Добавлено:

- честный экран блокировки `#lock-screen` с PIN из 4 цифр;
- хранение PIN как `SHA-256(pin + salt)` через Web Crypto (`stanza-diary.pin.v1`);
- блок «Приватность» в настройках: тумблер PIN, кнопка «сменить PIN»;
- информблок «где хранятся данные» с честным описанием локального хранилища;
- модал ввода PIN с точками-индикаторами и shake-анимацией ошибки;
- кнопка «сбросить PIN» на lock-screen с явным предупреждением, что данные не зашифрованы.

Изменено:

- `boot()` маршрутизирует по наличию PIN: есть PIN → lock-screen, нет → сразу дневник.

Удалено:

- бутафорский email/Google login (`#login-screen`, `auth-form`, `google-mark`, `divider`);
- ключ `stanza-diary.auth.v1` (стирается при старте через миграцию);
- кнопка «выйти» из настроек.

Отложено:

- автоблокировка после бездействия (этап 4 пункт 3);
- шифрование самих записей через Web Crypto (после IndexedDB);
- AI-агенты, внешние ключи.

## 0.1.0 - 2026-05-16

Первая рабочая версия.

Добавлено:

- статус автосохранения с состояниями `сохраняю`, `сохранено`, `ошибка`;
- шаблон дня для пустых полей записи;
- недельный обзор по локальным данным без AI;
- мобильный интерфейс по референсам Stanza Terminal;
- экран входа;
- экран записи;
- календарь месяца;
- история записей;
- статистика;
- настройки;
- редактирование записей;
- импорт и экспорт JSON;
- локальное хранение записей;
- PWA-заготовка: `manifest.webmanifest`, `sw.js`, `icon.svg`;
- модальное подтверждение вместо системного `confirm`;
- файл идей `docs/IDEAS.md`;
- план развития `ROADMAP.md`;
- описание архитектуры `docs/ARCHITECTURE.md`;
- модель данных `docs/DATA_MODEL.md`;
- архитектурные решения `docs/DECISIONS.md`;
- план внедрения `docs/IMPLEMENTATION_PLAN.md`;
- модель приватности `docs/PRIVACY.md`;
- `package.json` со скриптами запуска и проверки;
- README проекта.

Отложено:

- AI-агенты;
- подключение внешних ключей;
- локальное шифрование;
- полноценное IndexedDB-хранилище.
