# Decisions

Короткие архитектурные решения проекта.

## 001. Пока остаёмся на Vanilla JS

Дата: 2026-05-16

Решение: не переносить проект сразу на React/Vite-приложение.

Причины:

- текущий проект уже работает как статическое приложение;
- интерфейс пока лучше развивать отдельно через референсы;
- ближайший выигрыш даст модульность, а не смена фреймворка;
- vanilla-подход проще для PWA и локального запуска на старте.

Последствия:

- сначала переходим на ES modules;
- затем выделяем доменную логику, UI-рендеры и storage;
- к React или другому фреймворку можно вернуться позже, если проект перерастёт текущую модель.

## 002. AI-агенты откладываются

Дата: 2026-05-16

Решение: этап AI-агентов не входит в текущий roadmap.

Причины:

- для AI нужны ключи и политика безопасности;
- дневник содержит личные данные;
- сначала нужны приватность, хранение, экспорт и архитектура.

Последствия:

- этапы 1–6 развиваются без внешних AI API;
- локальная автоматизация разрешена;
- AI-функции хранятся как идеи в `docs/IDEAS.md`.

## 003. Storage сначала через repository

Дата: 2026-05-16

Решение: перед переходом на IndexedDB добавить единый интерфейс хранилища.

Причины:

- UI не должен зависеть от конкретного способа хранения;
- миграция из `localStorage` станет безопаснее;
- тестировать слой данных будет проще.

Последствия:

- будущие модули должны работать через `storage/repository.js`;
- IndexedDB adapter добавляется без переписывания экранов;
- импорт/экспорт остаются отдельной частью storage-слоя.
