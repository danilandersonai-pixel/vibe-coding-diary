# Privacy

Дневник содержит личные записи, поэтому приватность должна быть отдельным направлением проекта.

## Текущее состояние

Сейчас приложение работает локально в браузере.

Важно:

- записи не отправляются на сервер;
- данные лежат в браузерном хранилище;
- экран входа пока не является настоящей защитой;
- экспорт JSON создаёт незашифрованную копию данных.

## Ближайшая честная модель

Нужно явно объяснить пользователю:

- где хранятся записи;
- что удаление браузерных данных может стереть дневник;
- что JSON-экспорт нужно хранить аккуратно;
- что текущий вход не заменяет PIN или шифрование.

## План приватности

1. Добавить информационный блок в настройки.
2. Добавить локальный PIN-код.
3. Добавить автоблокировку.
4. Скрывать записи на заблокированном экране.
5. После IndexedDB добавить локальное шифрование через Web Crypto.
6. Добавить зашифрованный экспорт.

## Правило для AI

AI-функции нельзя подключать, пока не решены:

- управление ключами;
- согласие пользователя на обработку записей;
- режим локальной/внешней обработки;
- экспорт и удаление данных.
