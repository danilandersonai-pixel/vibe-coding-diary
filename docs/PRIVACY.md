# Privacy

Дневник содержит личные записи, поэтому приватность должна быть отдельным направлением проекта.

## Текущее состояние (0.2.0)

Приложение работает локально в браузере.

Что есть:

- записи не отправляются на сервер;
- PIN-блокировка из 4 цифр, хеш `SHA-256(pin + salt)` через Web Crypto;
- информблок в настройках честно объясняет, что это UX-блокировка, а не шифрование;
- бутафорский экран входа удалён.

Чего ещё нет:

- сами записи в `localStorage` не зашифрованы;
- нет автоблокировки по таймауту;
- JSON-экспорт по-прежнему незашифрован;
- кнопка «сбросить PIN» удаляет PIN без проверки — это сознательно, потому что данные и так не защищены криптографически.

## Правило для AI

AI-функции нельзя подключать, пока не решены:

- управление ключами;
- согласие пользователя на обработку записей;
- режим локальной/внешней обработки;
- экспорт и удаление данных.
