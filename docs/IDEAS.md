# Ideas

Этот файл хранит идеи развития проекта, чтобы они не потерялись между итерациями.

## Локальная автоматизация без AI-ключей

Цель: сделать дневник более живым и полезным без подключения внешних сервисов.

Возможные функции:

- статус автосохранения: `сохраняю`, `сохранено`, `ошибка`;
- умный шаблон дня для пустой записи;
- недельный обзор по локальным данным;
- автоматические подсказки для рефлексии;
- напоминание о резервной копии;
- восстановление черновика после перезагрузки;
- правила по тегам, например отдельные вопросы для `работа`, `учёба`, `здоровье`;
- локальные уведомления в PWA-режиме.

Первый практический шаг:

1. Добавить статус автосохранения.
2. Добавить шаблон дня.
3. Добавить недельный обзор.

## Будущие AI-агенты

AI-часть лучше подключать позже, когда будут понятны структура данных, приватность и способ хранения ключей.

Возможные направления:

- автоитоги недели;
- смысловой поиск по записям;
- чат по дневнику;
- подсказки для рефлексии;
- агент, который раз в неделю собирает обзор;
- агент, который предлагает темы и вопросы на основе прошлых записей.

Важное ограничение:

- не подключать AI-ключи, пока в проекте нет понятной схемы приватности и настроек безопасности.

## Продуктовые идеи

- полноценный календарный экран;
- группировка истории по месяцам;
- настроение, энергия, фокус и стресс дня;
- избранные записи;
- режим фокуса для письма;
- настраиваемые шаблоны;
- PIN-код или локальная блокировка;
- локальное шифрование записей;
- импорт и экспорт резервных копий.

## Технические идеи

- разложить `app.js` на модули;
- перейти с `localStorage` на IndexedDB;
- добавить тесты;
- настроить локальный сервер для PWA;
- подготовить деплой на GitHub Pages или Netlify;
- завести `ROADMAP.md`, `CHANGELOG.md` и `docs/ARCHITECTURE.md`.
