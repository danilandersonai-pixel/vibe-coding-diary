# Architecture

Документ описывает текущее устройство проекта и направление архитектурного роста.

## Текущее состояние

Проект пока устроен как простое статическое приложение.

Файлы:

- `index.html` — вся разметка экранов.
- `styles.css` — вся визуальная система и адаптация.
- `app.js` — состояние, сохранение, календарь, история, статистика, импорт/экспорт и настройки.
- `manifest.webmanifest` — PWA-описание.
- `sw.js` — service worker.
- `icon.svg` — иконка приложения.
- `reference/` — референсы интерфейса.
- `docs/` — проектная документация.

## Текущая модель данных

Записи хранятся в `localStorage` по ключу `stanza-diary.entries.v1`.

Пример записи:

```json
{
  "2026-05-16": {
    "date": "2026-05-16",
    "did": "Текст блока",
    "learned": "Текст блока",
    "result": "Текст блока",
    "failed": "Текст блока",
    "tags": ["пример"],
    "updatedAt": "2026-05-16T17:00:00.000Z"
  }
}
```

Настройки хранятся отдельно по ключу `stanza-diary.settings.v1`.

## Ограничения текущей архитектуры

- `app.js` слишком большой и совмещает много ответственностей.
- `localStorage` не подходит для долгого роста проекта.
- Нет тестов.
- Нет схемы валидации импортируемых данных.
- PWA работает полноценно только через локальный сервер или деплой.
- Нет приватности и блокировки.

## Желаемая модульная структура

На этапе 6 проект стоит перейти на ES modules без обязательной смены фреймворка. В `index.html` будущий вход будет выглядеть так:

```html
<script type="module" src="src/main.js"></script>
```

## Архитектурные правила

- UI не должен напрямую знать детали хранилища.
- Импорт данных должен проходить через нормализацию и проверку схемы.
- Автоматизация должна работать поверх публичных функций состояния, а не читать DOM напрямую.
- Приватность должна быть отдельным слоем, чтобы позже можно было добавить шифрование.
- AI-агенты нельзя подключать до появления приватности и понятного управления ключами.

## Ближайшая архитектурная цель

Сначала не нужно сразу переносить проект на React или другой фреймворк. Достаточно сохранить текущий vanilla-подход, но разнести код по модулям и добавить dev-сервер.
